üîß Backend Requirements

Note: The widget expects these backend endpoints:
  • POST /v1/widget/session - Create session (returns session_id, token)
  • POST /v1/widget/message - Send message (requires Authorization: Bearer token)

If these endpoints don't exist, the widget will fallback to using the /query/ endpoint.

Important: This widget does NOT use the /autoconnect endpoint (that's for the old iframe approach). The widget communicates directly with the API server via REST endpoints.

‚ö†Ô∏è Troubleshooting

404 Error on /autoconnect?

If you see a 404 error for /autoconnect, you're likely using the old iframe-based code. The new widget doesn't use that endpoint.

  • The new widget uses REST API endpoints (/v1/widget/session and /v1/widget/message)
  • It doesn't require the Flutter chatbot app to be running
  • It communicates directly with the API server
  • Make sure your data-api-base points to the API server (e.g., https://api.aventora.ai or http://localhost:8009)
